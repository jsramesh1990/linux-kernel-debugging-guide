# Introduction

Linux provides several tracing frameworks for debugging and performance analysis. While **ftrace** offers powerful kernel tracing capabilities, manually interacting with files inside:

```text
/sys/kernel/debug/tracing/
```

can become complex.

**trace-cmd** is a user-friendly command-line utility that simplifies the process of configuring, collecting, and analyzing ftrace data.

In Yocto and Embedded Linux development, trace-cmd is widely used for debugging kernel behavior, analyzing scheduling issues, measuring latency, and investigating driver problems.

---

# What is trace-cmd?

**trace-cmd** is a command-line tool that acts as a front-end for the Linux ftrace framework.

It simplifies:

* Trace collection
* Event management
* Trace recording
* Trace analysis
* Trace reporting

Instead of manually writing to ftrace files, developers can use simple trace-cmd commands.

---

# Why Use trace-cmd?

Developers use trace-cmd to:

* Record kernel events
* Analyze scheduler activity
* Debug device drivers
* Investigate latency problems
* Capture boot-time traces
* Analyze interrupts
* Monitor system behavior
* Simplify ftrace usage

# Relationship with ftrace

```text
Application
      │
      ▼
trace-cmd
      │
      ▼
ftrace Framework
      │
      ▼
Linux Kernel
```

Important:

* ftrace = Kernel tracing framework
* trace-cmd = User-space tool for managing ftrace

trace-cmd does not replace ftrace.

It provides a simpler interface to ftrace.

---

# How trace-cmd Works

```text
Enable Events
      ↓
Record Trace Data
      ↓
Store trace.dat
      ↓
Analyze Results
      ↓
Generate Report
```

Trace data is saved in:

```text
trace.dat
```

which can later be analyzed or shared.


Verify installation:

# Common Commands

## Record Kernel Events

```bash
trace-cmd record
```

Output:

```text
trace.dat
```

Generated automatically.

---

## Record a Specific Command

```bash
trace-cmd record ./application
```

Records kernel events generated by the application.

---

## Generate Report

```bash
trace-cmd report
```

Reads:

```text
trace.dat
```

and displays trace information.

---

## List Available Events

```bash
trace-cmd list
```

Displays supported tracing events.

---

## Record Scheduler Events

```bash
trace-cmd record -e sched
```

Captures:

* Context switches
* Task scheduling
* Wakeup events

---

## Record IRQ Events

```bash
trace-cmd record -e irq
```

Captures interrupt activity.

---

## Record Function Tracing

```bash
trace-cmd record -p function
```

Uses ftrace function tracer.

---

## Record Function Graph

```bash
trace-cmd record -p function_graph
```

Captures:

* Function entry
* Function exit
* Execution duration

---

## Follow Live Events

```bash
trace-cmd stream
```

Displays events in real time.

---

## Reset Tracing

```bash
trace-cmd reset
```

Clears tracing configuration.

# Understanding Trace Data

Typical report output:

```text
bash-1234 [001] 100.123456:
sched_switch
```

Breakdown:

### Process

```text
bash
```

Task generating the event.

### PID

```text
1234
```

Process ID.

### CPU

```text
[001]
```

CPU core.

### Timestamp

```text
100.123456
```

Event occurrence time.

### Event

```text
sched_switch
```

Kernel event name.

# trace-cmd vs ftrace

| Feature               | trace-cmd | ftrace  |
| --------------------- | --------- | ------- |
| User Interface        | Yes       | Manual  |
| Event Recording       | Yes       | Limited |
| Trace File Management | Yes       | No      |
| Function Tracing      | Yes       | Yes     |
| Kernel Framework      | No        | Yes     |

Summary:

* ftrace = Tracing framework
* trace-cmd = Tool to control ftrace

# Best Practices

* Record only required events
* Keep trace sessions short
* Use filters to reduce trace size
* Analyze traces offline when possible
* Combine with perf for performance investigations
* Reset tracing after use

Example:

```bash
trace-cmd reset
trace-cmd record -e sched
trace-cmd report
```

# Conclusion

trace-cmd is a powerful user-space utility that simplifies interaction with the Linux ftrace framework. It enables developers to record, store, and analyze kernel trace data efficiently. In Yocto and Embedded Linux environments, trace-cmd is commonly used for scheduler analysis, interrupt debugging, driver development, latency measurements, and kernel performance investigations.

### Quick Interview Answer

> **trace-cmd** is a user-space tool that provides an easy interface to the Linux **ftrace** framework. It is used to record and analyze kernel events such as scheduling, interrupts, and function calls. Trace data is stored in `trace.dat`, which can later be examined using `trace-cmd report`. It is widely used in Yocto and Embedded Linux systems for kernel debugging and performance analysis.
