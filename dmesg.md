# Introduction

When working with Linux-based embedded systems, one of the first commands engineers use for debugging is `dmesg`.

The `dmesg` command displays messages generated by the Linux kernel during:

* System boot
* Driver initialization
* Hardware detection
* Module loading
* Runtime kernel events

In Yocto-based embedded Linux systems, `dmesg` is an essential debugging tool for kernel, driver, and hardware-related issues.

---

# What is dmesg?

`dmesg` stands for:

```text
Display Message
```

It is a Linux command used to read and display the **kernel ring buffer**.

The kernel stores important system messages in memory, and `dmesg` allows users to view those messages.

---

# Why Use dmesg?

Developers use `dmesg` to:

* Check boot logs
* Verify hardware detection
* Debug device drivers
* Analyze kernel errors
* Troubleshoot boot failures
* Monitor kernel warnings
* Verify module loading

---

# How dmesg Works

Boot Flow:

```text
Power ON
    ↓
Bootloader
    ↓
Linux Kernel Starts
    ↓
Kernel Generates Messages
    ↓
Messages Stored in Ring Buffer
    ↓
dmesg Displays Messages
```

The kernel continuously writes messages to a circular memory buffer called the **kernel ring buffer**.

# Common Commands

## Display Complete Kernel Log

```bash
dmesg
```

Example:

```text
[    0.000000] Linux version 6.x
[    0.123456] CPU: ARM Cortex-A53
[    1.234567] EXT4-fs mounted
```

---

## Human Readable Timestamps

```bash
dmesg -T
```

Output:

```text
[Mon Jun  1 10:30:12 2026]
```

---

## Show Only Errors

```bash
dmesg --level=err
```

---

## Show Warnings

```bash
dmesg --level=warn
```

---

## Monitor New Messages

```bash
dmesg -w
```

Similar to:

```bash
tail -f
```

for kernel logs.

---

## Search for USB Devices

```bash
dmesg | grep USB
```

---

## Search for Ethernet Driver

```bash
dmesg | grep eth
```

---

## Search for Errors

```bash
dmesg | grep error
```

---

## Search for Driver Messages

```bash
dmesg | grep driver
```

# Understanding dmesg Output

Example:

```text
[    2.345678] mmc0: new high speed SDHC card
```

Breakdown:

```text
[Timestamp] Message
```

### Timestamp

```text
2.345678
```

Seconds after kernel boot.

### Message

```text
mmc0: new high speed SDHC card
```

Kernel event description.

# Advantages

## Quick Debugging

Provides immediate kernel information.

---

## Hardware Verification

Confirms hardware detection during boot.

---

## Driver Troubleshooting

Helps identify driver issues.

---

## Boot Analysis

---

# Disadvantages

## Large Output

Can be difficult to navigate on large systems.

---

## Ring Buffer Overwrites

Older messages may be lost.

---

## Root Access Restrictions

Some systems restrict kernel log access.

---

## Limited Historical Logs

Shows current ring buffer contents only.

---

# Best Practices

* Use `dmesg -T` for readable timestamps
* Filter output with `grep`
* Check logs immediately after hardware events
* Save logs for analysis:

```bash
dmesg > bootlog.txt
```

* Monitor live events using:

```bash
dmesg -w
```

* Combine with `journalctl` for complete diagnostics

---

# dmesg vs journalctl

| Feature              | dmesg   | journalctl |
| -------------------- | ------- | ---------- |
| Kernel Logs          | Yes     | Yes        |
| User Space Logs      | No      | Yes        |
| Boot Messages        | Yes     | Yes        |
| Historical Logs      | Limited | Yes        |
| Real-Time Monitoring | Yes     | Yes        |
